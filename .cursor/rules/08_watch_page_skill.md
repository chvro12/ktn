# Watch page — niveau réel attendu

La page watch est la page la plus importante.

## Elle doit inclure

- player principal sticky ou stable,
- titre vidéo,
- actions viewer (like, share, save, clip plus tard si besoin),
- infos chaîne + bouton subscribe,
- description expandable,
- commentaires,
- colonne de recommandations,
- mini-player ou queue si activé,
- autoplay toggle,
- progression sauvegardée.

## Exigences UX

- le player doit être crédible,
- commandes overlay sobres,
- bon comportement keyboard,
- bonne gestion du theater mode,
- plein écran propre,
- lecture HLS,
- reprise de lecture,
- tracking de progression,
- impression produit réel, pas démo Dribbble.

## À prévoir dans le player

- play/pause
- seek
- volume/mute
- quality selector
- speed selector
- captions toggle
- fullscreen
- theater mode
- timestamps preview si possible plus tard
