# Critères de qualité / Definition of Done

Une fonctionnalité n’est pas considérée terminée si elle n’a pas :

- un vrai rôle métier,
- un état loading,
- un état vide,
- un état erreur,
- une gestion responsive,
- des composants propres,
- une logique backend cohérente,
- un minimum de validation,
- un minimum d’accessibilité,
- un code maintenable.

Pour les flows critiques, il faut aussi :

- gestion des permissions,
- gestion des statuts,
- gestion des edge cases,
- instrumentation minimale,
- comportement stable sur refresh.
